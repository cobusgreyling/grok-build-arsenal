# browser-qa MCP

Visual + accessibility QA during UI work. Screenshots, console capture, a11y checks, diffing against previous states.

**Sole author: Cobus Greyling**

Intentionally the lightest MCP in the set — it exists to be called from `tdd-intelligence` or UI-focused arenas when you are changing frontends.

See `skills/tdd-intelligence/SKILL.md` for the recommended pairing ("When editing UI or complex flows, combine with `browser-qa`...").

A minimal server.py will be added when a concrete UI showcase needs it. For now the pattern + driving skill is what matters.

This MCP was designed using the arsenal discipline.
